## What This Project Is

A **free, fully static web dashboard** that shows which Indian banks are most/least reliable on UPI, using NPCI's monthly per-bank Technical Decline (TD%) data. No backend, no database, no paid API, no credit card. Deploys to Cloudflare Pages (or any static host).

**Core insight:** NPCI publishes two failure types per bank per month:
- **Technical Decline (TD%)** — the bank's/NPCI's own system failed. Not the user's fault. This is the ranking metric.
- **Business Decline (BD%)** — user-side failure (wrong PIN, low balance, daily limit). Excluded from ranking.

By stacking 18 months of TD data we separate *chronically weak* banks from *one-off outage* months, and weight by transaction volume to show whose failures actually affect the most people.

## Data Pipeline — How It Works

### Flow
```
seed_demo.py  →  pipeline/data/YYYY-MM.csv  (×18)
                        ↓
               ingest.py → tidy.csv
                        ↓
           build_json.py → web/public/data/banks.json
                         → web/public/data/meta.json
```

### `seed_demo.py`
Generates 18 monthly CSVs (2024-09 through 2026-02) with 12 rows each. Bank personalities:

| Bank | Role | Personality | Base TD% |
|---|---|---|---|
| HDFC Bank | remitter | rock_solid | ~0.30 |
| ICICI Bank | remitter | rock_solid | ~0.45 |
| Axis Bank | remitter | rock_solid | ~0.55 |
| State Bank of India | remitter | moderate (huge volume) | ~0.85 |
| Punjab National Bank | remitter | chronic | ~2.10 |
| Bank of Baroda | remitter | chronic | ~1.80 |
| Canara Bank | remitter | chronic | ~1.60 |
| Kotak Mahindra Bank | remitter | **one_off** (spike at month index 10 = 2025-07, TD ~6–9.5%) | ~0.50 |
| IndusInd Bank | remitter | volatile | ~1.10 ±0.55 |
| HDFC Bank | beneficiary | rock_solid | ~0.25 |
| State Bank of India | beneficiary | moderate | ~0.90 |
| Punjab National Bank | beneficiary | chronic | ~2.20 |

Columns generated: `Bank, Role, Approved %, Business Decline %, Technical Decline %, Volume (Mn)`

### `ingest.py`
- Parses month from filename via regex: `YYYY-MM.csv` or `Aug-2025.xlsx`
- Normalizes headers via editable `HEADER_ALIASES` dict at top of file — **edit this when NPCI changes column names**
- Cleans `%` strings → floats
- Outputs tidy table: `month, bank, role, approved_pct, bd_pct, td_pct, volume_mn`
- Saves to `tidy.parquet` (if pyarrow installed) or `tidy.csv` fallback

### `build_json.py`
Classification logic (in priority order):
1. `rock_solid`: mean_td < 0.6 **and** std_td < 0.4
2. `chronically_weak`: mean_td ≥ 1.5 **and** std_td < 1.0
3. `one_off_incident`: mean_td < 1.5 **but** max_td ≥ 3× mean_td **and** max_td > 4
4. `volatile`: std_td ≥ 0.4 (high variance)
5. Fallback: mean_td < 1.0 → rock_solid, else chronically_weak

Additional computed fields:
- `impact_score = mean_td / 100 × mean_volume_mn` (failed transactions in millions/month)
- `trend`: slope from `np.polyfit(x, td_vals, 1)` → "improving" / "worsening" / "flat" (|slope| < 0.01)
- `latest_td`: last month's TD%

## Frontend Architecture

### State management
All shared state lives in `App.tsx` and is passed as props to pages:
- `role` — "remitter" | "beneficiary"
- `sortKey` — "worst_sustained" | "worst_single" | "biggest_impact"
- `search` — string filter on bank name
- `monthRange` — [startIndex, endIndex] into allMonths array
- `selectedBank` — currently focused BankEntry (used by BankDetail)
- `compareList` — array of bank names for Compare view (max 3)

`banks` prop passed to pages is **pre-filtered** by role + search + monthRange + sorted by sortKey.

### Routing
```
/              → Overview (heatmap + leaderboard)
/map           → ReliabilityMap (scatter plot)
/bank/:bankName?role=remitter → BankDetail
/compare       → Compare
/about         → About/Methodology
```

### Design tokens (Tailwind)
```
bg-bg-primary     #0d1117   — page background
bg-bg-secondary   #161b22   — inputs, buttons
bg-bg-card        #1c2230   — cards
accent-cyan       #00e5ff   — THE accent (one color rule)
reliability-green #00c853   — TD < 0.5%
reliability-amber #ffab00   — TD < 1.5%
reliability-red   #ff1744   — TD ≥ 3%
text-primary      #e6edf3
text-secondary    #8b949e
text-muted        #484f58
```

### Color-blind safety strategy
Every color is **always paired** with:
- A shape/icon: ◆ (rock_solid) · ▲ (chronically_weak) · ● (one_off_incident) · ◇ (volatile)
- A numeric value (TD%)
- A text label ("Excellent / Good / Fair / Poor / Critical")

Never color alone anywhere in the UI.

### Heatmap implementation
Custom SVG in `Heatmap.tsx` (not a chart library). Key decisions:
- Rows = banks sorted worst→best (pre-sorted from JSON)
- Cols = months in chronological order
- Cell color: `tdColor()` from `lib/data.ts` (5-stop green→red scale)
- Cell text: TD% value in white (if dark cell) or dark (if light cell)
- Staggered fade-in via Framer Motion (`delay = (bankIdx * monthCount + monthIdx) * 0.002`)
- Horizontally scrollable container on mobile
- Full ARIA: `role="row"`, `role="cell"`, `aria-label` on every cell with bank + month + value
- Click/Enter on row navigates to BankDetail

## How to Swap in Real NPCI Data

1. Download the monthly bank-wise UPI data from NPCI's website.
2. Rename the file to `YYYY-MM.csv` (e.g. `2025-09.csv`).
3. Drop it into `pipeline/data/`.
4. If NPCI column names differ, update `HEADER_ALIASES` in `pipeline/ingest.py`.
5. Re-run:
   ```bash
   cd pipeline
   python ingest.py
   python build_json.py
   ```
6. Commit `web/public/data/banks.json` and `web/public/data/meta.json`.
7. Push → Cloudflare Pages auto-redeploys.

The GitHub Actions workflow (`.github/workflows/refresh-data.yml`) automates this on the 5th of each month if real CSVs are present in `pipeline/data/`.
